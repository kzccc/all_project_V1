# 五节点 Kafka 部署计划

## 1. 目标

这次要把当前单 broker 的 Kafka，升级成：

1. `5` 个 Kafka 节点
2. 单机本地可拉起
3. 项目可直接连通
4. 支持后续正式压测

## 2. 这次的部署策略

这次采用：

1. Kafka KRaft 模式
2. `5` 个 broker/controller 合并节点
3. 本机 `5` 进程本地部署为主，Docker Compose 作为备用方案

这样做的原因：

1. 不需要额外部署 ZooKeeper
2. 当前机器 Docker Hub 拉取超时，本机 5 进程方案可以直接落地
3. 机器上已有旧 Kafka 占用 `9092-9093`，新集群改到独立端口更稳
4. 后续压测切换成本最低

## 3. 这次的参数选择

### Kafka 集群参数

1. broker 数：`5`
2. chat topic 默认分区数：`240`
3. chat topic 副本因子：`3`
4. `min.insync.replicas = 2`

### 项目配置

新增单独配置：

1. [config_local_kafka5.toml](/workspace/czk/Personal/EchoChat/configs/config_local_kafka5.toml)

作用是：

1. 把 `messageMode` 切到 `kafka`
2. 把 broker 地址改成 `127.0.0.1:29092 ~ 29096`
3. 把 chat topic 切到独立名字 `chat_message_cluster5`
4. 把 topic 副本因子切到 `3`

## 4. 执行步骤

1. 先补应用侧 `topicReplicationFactor` 配置，避免程序建 topic 时仍然写死成单副本
2. 再落 Docker Compose 和本地 5 进程两套部署文件
3. 由于旧 Kafka 占用 `9092-9093`，本次本地集群统一改到 `29092-29096`
4. 启动 5 节点 Kafka 集群
5. 验证 broker 连通、quorum 状态和 topic 创建能力
6. 切到 5 节点配置后，准备正式压测
