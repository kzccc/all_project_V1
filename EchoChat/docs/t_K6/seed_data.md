# 压测测试数据生成说明

## 1. 目标

这套测试数据生成器用于在 EchoChat 本地环境中快速生成一批可重复、可清理、可扩量的压测数据，覆盖以下几类核心业务对象：

- 用户
- 好友关系
- 群组
- 群成员关系
- 会话
- 历史消息
- 联系人申请

它的目标不是模拟线上所有复杂行为，而是给后续压测提供一批结构稳定、规模可控、可以反复重建的数据底座。

## 2. 生成器位置

生成器代码：

- `cmd/echo_chat_seed/main.go`

推荐启动脚本：

- `docs/t_K6/scripts/seed_testdata.sh`

执行记录输出目录：

- `docs/t_K6/records`

## 3. 默认生成规模

脚本默认会生成如下规模的数据：

- 200 个测试用户
- 1 个管理员账号
- 12 个测试群组
- 每个群 25 个成员
- 每个用户与后续 10 个用户建立好友关系
- 每个好友对生成 30 条单聊消息
- 每个群生成 120 条群聊消息
- 40 条待处理联系人申请

这套默认规模适合作为第一轮压测前的数据准备基线。

## 4. 数据命名规则

为了便于清理和重复执行，生成器默认使用统一前缀：

- 前缀默认值：`K6`

生成出的业务 ID 会带固定前缀：

- 用户：`UK6...`
- 群组：`GK6...`
- 会话：`SK6...`
- 消息：`MK6...`
- 联系人申请：`AK6...`

测试账号默认密码：

- `123456`

测试账号手机号默认从以下区间开始连续生成：

- `17610000001`

## 5. 执行方式

### 5.1 用推荐脚本执行

```bash
bash /workspace/czk/Personal/EchoChat/docs/t_K6/scripts/seed_testdata.sh
```

### 5.2 直接运行 Go 生成器

```bash
cd /workspace/czk/Personal/EchoChat
go run ./cmd/echo_chat_seed \
  -prefix K6 \
  -reset-prefix \
  -user-count 200 \
  -admin-count 1 \
  -group-count 12 \
  -group-size 25 \
  -friend-span 10 \
  -pair-messages 30 \
  -group-messages 120 \
  -apply-count 40 \
  -password 123456 \
  -telephone-start 17610000000
```

## 6. 参数说明

| 参数 | 含义 |
| --- | --- |
| `-prefix` | 本次生成数据的业务前缀 |
| `-reset-prefix` | 生成前是否先清理同前缀旧数据 |
| `-user-count` | 生成用户数量 |
| `-admin-count` | 管理员用户数量 |
| `-group-count` | 群组数量 |
| `-group-size` | 每个群的成员数量 |
| `-friend-span` | 每个用户向后建立多少个好友关系 |
| `-pair-messages` | 每对好友生成多少条单聊消息 |
| `-group-messages` | 每个群生成多少条群消息 |
| `-apply-count` | 生成多少条待处理联系人申请 |
| `-password` | 测试账号密码 |
| `-telephone-start` | 手机号起始值 |

## 7. 设计原则

这套数据生成器遵循以下原则：

1. 可重复执行
2. 同前缀数据可整体清理
3. 生成后的数据可直接用于 HTTP 和 WebSocket 压测
4. 会同步清理聊天相关缓存，避免旧缓存污染结果

## 8. 适用场景

这套数据适合直接支撑下面这些压测场景：

- 登录与刷新 token 压测
- 联系人列表、会话列表压测
- 历史消息查询压测
- WebSocket 建连压测
- 单聊消息实时性压测
- 群聊广播压测
- 长稳压测

## 9. 建议执行顺序

建议每次压测前按下面顺序准备环境：

1. 确认 `echochat.service` 正常运行
2. 执行测试数据生成脚本
3. 记录本次生成输出到 `docs/t_K6/records`
4. 再开始执行对应压测脚本

## 10. 说明

当前生成器优先服务于压测准备，因此更关注：

- 数据规模
- 关系完整性
- 查询可用性
- 实时链路可压测性

后续如果需要，我们可以继续扩展两类数据：

1. 文件消息、语音消息、通话信令消息
2. 更复杂的群申请、黑名单、禁用状态和异常数据样本
