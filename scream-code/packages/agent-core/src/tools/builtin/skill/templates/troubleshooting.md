# Skill type: troubleshooting

Focus on the symptoms, root causes, and resolution steps for a problem that was solved in the conversation. The goal is to create a diagnostic playbook.

The generated skill should include:
1. The observable symptom(s) this playbook addresses.
2. Likely root causes, ordered from most to least common.
3. A diagnostic sequence: which files/commands/logs to inspect, and what to look for.
4. The fix or workaround for each root cause.
5. How to verify the issue is resolved.
6. "Prevention" notes if applicable (configuration, tests, monitoring).

Use precise but reusable language; avoid hard-coding values that only apply to the current project.
