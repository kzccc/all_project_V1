# Skill type: code-pattern

Focus on the coding conventions, idioms, or architectural patterns that appeared in the conversation. The goal is to capture a reusable pattern the assistant can apply to similar code tasks.

The generated skill should include:
1. The situation where this pattern applies (language/framework/domain).
2. The invariant or design principle being enforced.
3. Concrete implementation guidance: preferred imports, function/class shapes, error handling, naming, etc.
4. One or two minimal, well-commented code examples that illustrate the pattern.
5. Common pitfalls or anti-patterns to avoid.
6. Verification steps (e.g. "run typecheck", "run the specific test", "lint affected files").

Keep examples short enough to fit in context; mark placeholders like `<ComponentName>` clearly.
