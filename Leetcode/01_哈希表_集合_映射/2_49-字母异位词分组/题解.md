# 49. 字母异位词分组

## 题目概述

给定一个字符串数组，把由相同字母重排得到的字符串放进同一组。

## 面试最佳解法

首选 `排序后字符串作为哈希 key`。

原因：

1. 这是最常见、最好讲清楚的写法。
2. 相同字符组成的字符串，排序后结果一定完全相同。
3. 代码量小，稳定，不容易在字符计数细节上出错。

## 核心思路

对每个字符串：

1. 把字符排序。
2. 排序后的结果作为哈希表 key。
3. 原字符串追加到对应分组里。

最后把哈希表里的所有 value 收集起来即可。

## 关键不变量

如果两个字符串互为字母异位词，那么它们排序后的字符串一定相同；反过来也成立。

## 复杂度

- 时间复杂度：`O(n * k log k)`，`k` 是单个字符串长度
- 空间复杂度：`O(n * k)`

## 容易写错的点

1. 分组的 key 应该是排序后的结果，不是原字符串。
2. 要把原字符串放进答案，不是把排序后的 key 放进去。
3. 本地判题为了稳定输出，会在 `runner` 里做一次规范化排序，但核心解法不需要关心输出顺序。

## 备选方案

也可以用 `26` 个字母计数作为 key，这样能把单字符串复杂度降到 `O(k)`，但 key 构造更绕。面试里通常先写排序版更稳。
