# 239. 滑动窗口最大值

## 题目概述

给定一个数组 `nums` 和窗口大小 `k`，窗口从左到右每次移动一格。你需要在每个窗口里取出一个最大值，最终返回所有窗口最大值组成的数组。

例如：

```text
nums = [1,3,-1,-3,5,3,6,7], k = 3
```

窗口依次是：

```text
[1,3,-1] -> 3
[3,-1,-3] -> 3
[-1,-3,5] -> 5
[-3,5,3] -> 5
[5,3,6] -> 6
[3,6,7] -> 7
```

答案就是：

```text
[3,3,5,5,6,7]
```

## 面试最佳解法

主解法是 `单调队列`。

原因很直接：

1. 暴力做法每个窗口都重新找最大值，时间复杂度是 `O(nk)`，窗口大时会超时。
2. 堆也能做，但要处理“堆顶元素已经滑出窗口”的延迟删除，讲起来和写起来都更绕。
3. 单调队列能做到 `O(n)`，而且非常符合这题“窗口滑动 + 始终维护最优候选”的结构，是面试里最稳的首选。

## 核心思路

队列里不直接存值，而是存 **下标**，并维护成“对应值单调递减”的状态。

这样做有两个好处：

1. 队头永远是当前窗口最大值的下标。
2. 通过下标可以判断某个元素是否已经滑出窗口。

每走到一个新位置 `i`，做三件事：

1. 先把队头里已经不在窗口范围内的下标弹出。
2. 再把队尾所有“小于等于当前值”的下标弹出，因为它们以后不可能成为最大值了。
3. 把当前下标 `i` 放进队尾。

当 `i >= k-1` 时，说明一个完整窗口形成了，此时队头就是这个窗口的最大值。

## 关键状态或不变量

- 队列中的下标始终属于当前窗口范围。
- 队列中对应的值始终单调递减。
- 队头下标对应的值始终是当前窗口最大值。

第三条是这题最核心的不变量。

## 复杂度

- 时间复杂度：`O(n)`  
  每个元素最多进队一次、出队一次。
- 空间复杂度：`O(k)`  
  队列里最多只会保留当前窗口相关的下标。

## 容易写错的点

1. 队列里要存 **下标**，不是只存值，否则没法判断元素有没有滑出窗口。
2. 过期判断要写成 `deque[0] <= i-k`，表示它已经落在窗口左边界外。
3. 队尾弹出时通常写 `<=`，这样相同值时保留更靠右的那个下标，窗口寿命更长。
4. 结果收集从 `i >= k-1` 才开始，因为这时第一个完整窗口才出现。

## 备选方案

- `暴力`：每个窗口重新扫描最大值，`O(nk)`。
- `堆 / 优先队列`：时间复杂度一般写作 `O(n log k)`，可做，但实现和讲解都没有单调队列直接。
