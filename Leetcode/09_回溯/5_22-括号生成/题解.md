# 22. 括号生成

## 题目概述

给定 `n` 对括号，要求生成所有合法括号组合。

## 面试最佳解法

首选 `回溯 + 剪枝`。

因为这题不是先生成所有串再判合法，而是在构造过程中就保证合法性，面试里这是最标准的表达。

## 核心思路

维护两个计数：

- `open`：已经放了多少个左括号
- `close`：已经放了多少个右括号

规则：

1. `open < n` 时可以继续放左括号。
2. `close < open` 时才可以放右括号。

## 复杂度

答案规模是卡特兰数，回溯复杂度通常按答案规模理解。

## 容易写错的点

1. 忘记限制 `close < open`。
2. 结束条件写错，不是 `open == n && close == n` 之外的其他模糊写法。
3. 字节切片回退不及时。
