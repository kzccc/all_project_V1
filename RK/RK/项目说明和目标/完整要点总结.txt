一、项目定位

本项目是基于瑞芯微 RK3568 开发板的嵌入式视觉识别实训项目，核心目标是把“图像采集、LCD 显示、人机交互、摄像头控制、AI 推理识别、板端部署”串成一套可运行的端侧 AI 系统。

项目不只是算法演示，也不只是硬件驱动练习，而是围绕“算法模型 - NPU 硬件加速 - 嵌入式 Linux - 实际界面交互”的完整链路展开。最终应形成一个可在 RK3568 板端运行的视觉识别终端。

二、总体目标与评分点

1. 界面设计，20 分
   - 使用 LCD 屏幕显示项目界面。
   - 支持 BMP 图标显示。
   - 扩展方向：支持 JPG、PNG 图片显示。

2. 人机交互，20 分
   - 使用触摸屏获取触摸坐标。
   - 根据坐标判断触摸区域。
   - 支持点击按钮触发不同功能。
   - 扩展方向：支持滑屏、上一张/下一张图片切换等交互。

3. 摄像头数据采集，20 分
   - 能打开摄像头。
   - 能抓拍图片。
   - 能关闭摄像头。
   - 扩展方向：支持录制视频。

4. 视觉识别，20 分
   - 部署或调用推理模型。
   - 使用训练好的模型文件实现种类区分。
   - 扩展方向：数据标注、模型训练、模型转换、模型量化。

三、系统总体架构

系统以 RK3568 开发板为核心硬件，依托 Linux、FrameBuffer、V4L2、RKNN Runtime、RKNN Model Zoo、OpenCV、C/C++ 程序等组件实现。

核心模块包括：

1. 核心硬件模块
   - RK3568 开发板。
   - 内置 NPU，可用于端侧 AI 推理。
   - 支持 HDMI、MIPI-DSI、MIPI-CSI、USB、PCIe、千兆网口等接口。

   - 使用 USB 或 MIPI 摄像头。
   - 基于 V4L2 获取实时图像。
   - 完成摄像头打开、短探测、抓拍、关闭，后续可扩展录制。

3. 显示模块

   - Linux FrameBuffer 设备通常为 /dev/fb0。
   - 当前屏幕资料中给出的分辨率为 1024x600，像素格式按 32 位 ARGB 处理。

4. 图标与图片显示模块
   - 已整理 BMP 显示教程。
   - 可显示背景图、按钮图标、多张图标。
   - 后续扩展 JPG 显示，需要移植 libjpeg。

5. 触摸交互模块
   - 读取 /dev/input/eventX 输入设备。
   - 获取触摸坐标。
   - 判断坐标是否落入按钮区域。
   - 驱动“打开摄像头、关闭摄像头、抓拍、识别、上一张、下一张、退出”等功能。

6. AI 推理模块
   - 使用 RKNN Model Zoo 或训练好的模型文件。
   - 模型转换为 .rknn 后在板端通过 RKNN Runtime 调用 NPU 推理。
   - 支持目标检测、图像分类、人脸识别、语义分割等方向。

7. 通信与部署模块
   - 当前已搭建云服务器桥接通道。
   - 板子可从云服务器下载可执行文件并运行。

五、LCD 与 BMP 显示要点

1. LCD 硬件特性
   - LCD 对应 Linux 帧缓冲设备，常见路径为 /dev/fb0。
   - 截图资料中提到屏幕宽 1024、高 600。
   - 每个像素按 32 位处理，即 4 字节。
   - 像素属性通常由透明度、红、绿、蓝组成，即 ARGB。

2. 基础文件操作
   - open：打开 /dev/fb0 或 BMP 文件。
   - read：读取 BMP 文件头和像素数据。
   - write：可直接向 FrameBuffer 写入像素缓冲。
   - close：关闭设备或文件。
   - mmap：更推荐把 FrameBuffer 映射到用户空间后直接写显存。

3. BMP 文件结构
   - 文件头通常为 54 字节。
   - BITMAPFILEHEADER 14 字节，包含 BM 标识、文件大小、数据偏移等。
   - BITMAPINFOHEADER 40 字节，包含宽、高、位深、压缩方式等。
   - 24 位 BMP 的像素数据按 B、G、R 顺序存储。

4. BMP 到 LCD 的颜色转换
   - BMP 常见为 24 位 BGR。
   - LCD 当前按 32 位 ARGB 显示。
   - 需要把 b、g、r 组装成 color = 0xFF000000 | (r << 16) | (g << 8) | b。

5. BMP 垂直方向处理
   - BMP 默认常按自下而上存储。
   - LCD 通常按自上而下显示。
   - 显示时需要做垂直翻转，坐标可按 y = height - 1 - j 转换。

6. 多图显示
   - 建议封装通用函数，例如 lcd_draw_bmp(pathname, x, y, w, h) 或 show_bmp(filename, pos_x, pos_y, scale)。
   - 支持指定显示起点、图片大小、缩放比例。
   - 可先显示背景图，再叠加按钮图标。

7. 当前项目需要准备的图标
   - 打开摄像头。
   - 关闭摄像头。
   - 抓拍图片。
   - 录制视频，可选。
   - 视觉识别。
   - 上一张图片。
   - 下一张图片。
   - 退出图标。

六、JPG/JPEG 显示扩展要点

1. 显示原理
   - BMP 无压缩，可直接读取像素数据后显示，但文件大。
   - JPEG 有损压缩，体积小，但显示前需要先解码。
   - JPEG 显示流程：压缩数据 -> libjpeg 解压 -> RGB 数据 -> FrameBuffer 显示。

2. libjpeg 移植流程
   - 下载 jpegsrc 源码包。
   - 在 WSL 或 Linux 环境解压。
   - 进入源码目录后配置交叉编译参数。
   - 示例命令：
     tar zxvf jpegsrc.v9f.tar.gz -C ~
     cd ~/jpeg-9f
     ./configure --host=aarch64-linux --prefix=${HOME}/libjpeg
     make
     make install-strip
   - 安装后验证生成的库是否为 ARM 平台：
     file ${HOME}/libjpeg/lib/libjpeg.so.10.0.0

3. 打包上传
   - 可把 libjpeg 的 lib、bin 等目录打包后上传开发板。
   - 示例：
     tar zcvf libjpeg2arm.tar.gz lib bin

4. JPEG 解码代码要点
   - 包含 jpeglib.h、setjmp.h。
   - 使用 jpeg_decompress_struct 创建解码对象。
   - 设置错误处理。
   - 打开 JPEG 文件，读取头信息。
   - 获取 image_width、image_height。
   - 分配输出缓冲 out_buffer。
   - 逐行读取 scanline，并拷贝到完整 RGB 缓冲。
   - 将 RGB 缓冲转换成 LCD 像素格式后写入 FrameBuffer。

5. 编译与运行
   - 编译时需要指定头文件路径、库路径和 jpeg 库。
   - 示例：
     arm-linux-gcc pic.c -o pic -I ~/libjpeg/include -L ~/libjpeg/lib -ljpeg
   - 板端运行前复制图片和程序，赋予执行权限：
     chmod +x pic
     ./pic

6. 动态图片扩展
   - 可以连续显示多张图片并延时，形成简单动图效果。
   - 示例思路：lcd_draw_jpeg("1.jpg", 0, 0); usleep(1000); 循环刷新。

七、交叉编译与程序运行要点

1. 编码规范
   - C 源文件扩展名使用 .c。
   - C++ 源文件扩展名使用 .cpp。
   - C 头文件使用 .h。
   - C++ 头文件使用 .hpp。
   - 源码建议使用 UTF-8 编码。
   - Linux 下注意 LF 换行，避免 Windows CRLF 引发脚本或编译问题。

2. 基础交叉编译
   - 使用 aarch64-linux-gnu-gcc 将源文件编译为 ARM64 可执行程序。
   - 示例：
     aarch64-linux-gnu-gcc main.c -o main

3. 上传与赋权
   - 编译生成 main 后上传到开发板。
   - 运行前赋予执行权限：
     chmod +x main

4. 运行方式
   - 推荐使用相对路径：
     ./main
   - 也可使用绝对路径：
     /root/main

5. 避免默认程序占用屏幕
   - 如果板子启动后已有默认 UI 程序运行，可能遮挡自己的程序。
   - 可用命令结束默认演示进程：
     killall -9 rk356x-demo

十、AI 视觉识别实现要点

1. 模型来源
   - 优先使用 RKNN Model Zoo 中已适配 RK 平台的模型。
   - 可选模型类型：目标检测、图像分类、人脸识别、语义分割等。

2. 模型转换
   - 使用 RKNN-Toolkit2 将 PyTorch、TensorFlow、ONNX 等模型转换为 .rknn。
   - 支持 INT8 量化，降低模型体积并提高板端推理速度。

3. 板端推理
   - 使用 RKNN Runtime 加载 .rknn 模型。
   - 将摄像头或图片预处理后的输入数据送入 NPU。
   - 解析输出类别、位置、置信度。
   - 目标检测任务需要做后处理，例如非极大值抑制。

4. 与界面联动
   - 识别结果可以叠加到 LCD 显示画面。
   - 可根据识别结果触发蜂鸣器、继电器、指示灯等外设。
   - 项目当前优先完成“种类区分”和基础识别展示。

十四、风险与注意事项

1. 板端环境是 Buildroot，功能较精简，不能按完整 Ubuntu 使用。
2. 板端不要现场编译，优先云服务器交叉编译。
3. 操作摄像头前先短探测，避免程序长时间阻塞。
4. LCD 显示时要确认分辨率、色深和行跨度，不能只假设 width * height * 4。
5. BMP/JPG 图片的宽高、位深、方向需要处理，否则会出现花屏、倒置或颜色错误。
6. 触摸坐标可能需要校准，要根据实际屏幕方向做坐标映射。
7. 桥接心跳离线时，先确认板子电源、网络和自启客户端。
8. 不要改动现有 SSH 登录方式和桥接自启配置，避免影响后续远程开发。

十五、一句话总结

这个项目的核心是：在 RK3568 开发板上，用 LCD 显示图形界面，用触摸屏做人机交互，用摄像头采集图像，用 RKNN/NPU 做视觉识别，最后通过云端交叉编译和桥接通道完成板端部署与演示。
